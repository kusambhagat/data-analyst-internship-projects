# Sales Analysis Project

## Objective
Analyse routine retail sales records to understand monthly sales fluctuations, customer contribution to revenue, and product demand which supporting data-driven decisions on inventory and marketing.

## Data
Transactional sales dataset containing customer details, product info, price, and timestamps.

## Tools Used
Python (Pandas, Matplotlib, Seaborn)

## Approach
- Cleaned the dataset: checked data types, removed duplicates, handled missing values, extracted timestamp details
- Generated summary statistics (average quantity sold, price range, revenue spread)
- Built visualisations to identify sales trends and high-demand products

## Key Findings
- Average quantity sold per transaction: ~1.55 units (std dev 0.88) that's a narrow spread
- Price ranged from £20–£800 (avg ~£325.50)
- Revenue per transaction ranged £25–£1200 (std dev £390.42) — significant variation driven by product mix
